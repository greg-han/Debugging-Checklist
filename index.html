Your a new programer and your stuck.

Try these suggestions and see if you can get your project working.

These suggestions are independent of language. Each section ends with "Ask for help." If you ask for help too early it is hard to learn and gain confidence. If you wait to long you waist time and energy. I try to balance these two concerns. As long as you are making progress keep trying.

    Signs of progress are:
  • Knowing when the error occurs
  • Getting new error messages
  • Limiting where an error could be coming from.

Flow Chart For Complete Newbies on what to do when Code Doesn’t work

My Code Won’t Run / I got an error or other message - (usually Syntax errors or Compile Time errors):

    Syntax errors are the best kind of error to have. They always come with error messages. Error messages are kindly bud dumb genies telling you where your code is confusing the computer. (about half the time the computer is wrong about what should be fix)
    1. What to Look for in error messages:
    • What line in your program
    • What the computer wants / key words
    • Look for Red or messages
  • On your line? Syntax, number/type/order of arguments, wrong type of function for type/array (in weakly typed goes other place)
  • Before your line? Closing braces ";" initialization…
  • Proof read check spelling and variable names
  • Be your computer (sometimes your variables don't have what you think they do)
  • Google language + key word
  • Comment out your line
  • Retype / try your program line by line checking to see if it works
  • Close and reopen IDE, turn off computer
  • Ask for Help (get variations + explaining helps + get new error/problem + celebrate new line)

      My Code runs but I don’t understand what it is doing or it does nothing – (usually logic errors):

      Don't change things randomly it causes distractions / creating other error / version control

    1. Look for what line/s in your code created the output you see
    2. Be your computer
    3. Add logs/ break point in your debugger
    4. Comment out blocks of code
    5. Ask why is the computer putting this out?
    6. Look for flow control statements (loop, if …, switch) Check to see what parts of you code are running
    7. Retype / proofread your code
    8. Ask for help
